TIMESIGNAL.CONF(5) File Formats Manual TIMESIGNAL.CONF(5)

NAME

timesignal.conf - configuration file for timesignal(1)

SYNOPSIS

/etc/timesignal.conf

DESCRIPTION

timesignal(1) can obtain its options from a configuration file.

By default, it looks for this file at /etc/timesignal.conf, although this can be changed using the -C/--config command-line argument.

The format of the configuration file consists of a series of Option=Value assignments, one per line. This is similar to the format used in INI files, but even simpler in that [section] headers are not supported.

The example below demonstrates all the essential features of the format:

# Everything on a line after “#” is a comment.
# Comments and blank lines are ignored.
# Option assignments have an option name,
# “=” as a separator, and an option value.
Option=Value
# Values may be quoted in regular (non-curly) single or double quotes.
# They may also contain spaces, even if not quoted.
Option="Value with spaces"
Option='Value with spaces'
Option=Value with spaces
# A few options do not require specifying an option value.
# For such options, the value On is implied.
Option
Option=On
# Finally, option names and most option values
# (except for filenames) are not case-sensitive.
Option=Value
OPTION=VALUE
option=value

OPTIONS

Below are the options recognized in a configuration file, along with brief descriptions, allowed values, and defaults. Options require values unless otherwise specified. Some options default to special values that cannot be explicitly set in a configuration file.

Each configuration file option corresponds exactly to a command-line option. See timesignal(1) for more details than may be shown here.

Time station selection

station
Time station to emulate.
May be BPC, DCF77, JJY, JJY60, MSF, or WWVB.
Default is WWVB.

Time signal options

base
Time base.
Date and time in YYYY-MM-DD HH:mm:ss[(+-)hhmm] format, from 1970-01-01 00:00:00+0000 to 9999-12-31 23:59:59+2359.
Default is the current system time (special value).
offset
User offset.
Time quantity in [+-]HH:mm:ss[.SSS] format, from -23:59:59.999 to +23:59:59.999.
Default is 00:00:00.000.
dut1
DUT1 value (only for MSF and WWVB).
Number of milliseconds in [+-]SSS format, from -999 to +999.
Default is 0.

Timeout options

timeout
Time to run before exiting.
Time quantity in HH:mm:ss format, from 00:00:01 to 23:59:59.
Default is forever (special value).

Sound options (rarely needed)

method
Output method.
May be PipeWire, PulseAudio, or ALSA.
Default is autodetect (special value).
device
Output device (only for ALSA).
ALSA device name.
Default is default.
format
Output sample format.
May be S16, S16_LE, S16_BE, S24, S24_LE, S24_BE, S32, S32_LE, S32_BE, U16, U16_LE, U16_BE, U24, U24_LE, U24_BE, U32, U32_LE, U32_BE, FLOAT, FLOAT_LE, FLOAT_BE, FLOAT64, FLOAT64_LE, or FLOAT64_BE.
Default is S16.
rate
Output sample rate.
May be 44100, 48000, 88200, 96000, 176400, 192000, 352800, or 384000.
Default is 48000.
channels
Output channels.
Number of channels, from 1 to 1023.
Default is 1.
smooth
Smooth rapid gain changes in output waveform.
Does not require a value.
May be On, Off, or not provided (same effect as On).
Default is Off.
ultrasound
Enable ultrasound output (MAY DAMAGE EQUIPMENT).
Does not require a value.
May be On, Off, or not provided (same effect as On).
Default is Off.
audible
Make output waveform audible (for entertainment only).
Does not require a value.
May be On, Off, or not provided (same effect as On).
Default is Off.

Logging options

log
Log messages to a file.
Path to a file.
Default is none (special value).
syslog
Log messages to syslog.
Does not require a value.
May be On, Off, or not provided (same effect as On).
Default is Off.
verbose
Increase logging verbosity.
Does not require a value.
May be On, Off, or not provided (same effect as On).
Default is Off.
quiet
Suppress logging to console (and only console).
Does not require a value.
May be On, Off, or not provided (same effect as On).
Default is Off.
